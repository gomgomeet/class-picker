# 작업 저장 및 백업 (save-work)

아래 순서를 반드시 끝까지 수행한다.

## 1. 작업 내용 정리

이번 세션(또는 마지막 저장 이후)에 한 작업을 되돌아보고 정리한다:
- 무엇을 만들었나 / 수정했나
- 어떤 파일이 바뀌었나 (`git status`, `git diff --stat`으로 확인)
- 다음에 이어서 할 일은 무엇인가

## 2. PROGRESS.md 업데이트

저장소 루트의 `PROGRESS.md` 파일 **맨 위에** 새 항목을 추가한다 (없으면 새로 만든다).
형식:

```markdown
## YYYY-MM-DD (요일)

### 오늘 한 일
- ...

### 변경된 파일
- ...

### 다음에 할 일
- ...
```

날짜는 `date '+%Y-%m-%d'` 명령으로 확인한다. 한국어로 작성한다.

## 3. 커밋 & 푸시

```bash
git add -A
git commit -m "<작업 내용을 요약한 한국어 커밋 메시지>"
git push -u origin <현재 브랜치>
```

- 커밋 메시지는 "자동 백업" 같은 말 대신 실제 작업 내용을 요약해서 쓴다. (예: "수강신청 튜토리얼 2장 작성")
- push가 네트워크 오류로 실패하면 2초, 4초, 8초 간격으로 최대 4번 재시도한다.

## 4. 결과 보고

사용자에게 한국어로 보고한다:
- 저장된 파일 목록
- 커밋 메시지
- push 성공 여부 (GitHub에서 확인 가능하다는 것)
- PROGRESS.md에 기록된 "다음에 할 일"
