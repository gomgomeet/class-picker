# Inbox 음성 메모 요약·정리 스킬

곰곰 비서 앱(또는 직접 입력)으로 ✅ Inbox에 쌓인 항목 중 **요약이 비어 있는 것**을 찾아 요약을 채우고 분류를 제안한다.

## 전체 파이프라인 (확정 워크플로)

```
🎤 곰곰 비서 녹음 → 앱이 '인사이트'로 자동 분류 → CSV → ✅ Inbox(분류=인사이트)
→ [요약] 노션 AI 자동 채우기 또는 이 스킬이 요약
→ [액션 플랜] 이 스킬이 다이어리에 생성 + '관련 자료' 관계형으로 원본 연결
```

## 대상 DB

- **✅ Inbox** (2026 Dashboard): https://app.notion.com/p/652e8df29f3147609d597f9f53af7436
  - 데이터소스: `collection://b195b54c-77f8-4d9a-b38b-362ad33948a9`
  - 주요 필드: `제목`(title) · `요약`(text) · `분류`(select: 인사이트 등) · `출처`(select: voice/메신저/직접입력) · `수신일`(date)
- **📓 다이어리** (2026 Dashboard): https://app.notion.com/p/2b9ec7eb377a8181a7b1c4f9157679ab
  - 데이터소스: `collection://2b9ec7eb-377a-819a-855a-000ba78e980d`
  - 주요 필드: `이름`(title) · `선택`(select: 업무/개인/학년 등) · `상태`(status: 예정/진행 중/완료) · `날짜`(date) · `설명`(text) · **`관련 자료`(relation → ✅ Inbox)**

## 절차

1. **조회**: notion MCP `query-data-sources` 또는 데이터소스 SQL로 `요약`이 비었거나 짧은 항목을 가져온다. 음성 메모 우선: `출처 = 'voice'`. 사용자가 기간을 말하면 `수신일` 필터.
2. **요약 작성**: 각 항목의 본문·제목을 읽고 **2~3문장 요약 + 핵심 키워드**를 만든다.
   - 말로 받아쓴 글 특성(구어체·반복·오인식)을 감안해 매끄럽게 정리하되, 화자가 말한 사실을 지어내지 않는다.
3. **분류 제안**: 내용이 아이디어·배움이면 `분류=인사이트`, 할 일이면 `오늘시작`, 수업 자료면 `수업자료`. 이미 분류가 있으면 건드리지 않는다.
4. **업데이트**: `notion-update-page`로 `요약` 채우기(+미분류였다면 분류). 원문(제목/본문)은 수정하지 않는다.
   - 사용자가 노션 AI 자동 채우기를 켜둔 경우 요약이 이미 있을 수 있음 → 있으면 건너뛰고 액션 플랜 단계로.
5. **액션 플랜 → 다이어리 입력**: 각 인사이트의 요약에서 **실행할 일**을 뽑아
   📓 다이어리 DB에 항목 생성:
   - `이름` = 액션 한 줄 (동사로 시작: "~해보기", "~적용하기")
   - `상태` = `예정` · `날짜` = 오늘(또는 사용자가 말한 시점) · `선택` = 내용에 맞게(업무/개인/학년)
   - **`관련 자료` = 원본 ✅ Inbox 항목 URL** (관계형 연결 — 필수)
   - 실행할 일이 없는 순수 기록형 인사이트는 다이어리 생성 생략(억지로 만들지 않음)
   - 같은 원본에서 이미 만든 액션이 있으면 중복 생성하지 않음
5-1. **일정 감지 → 구글 캘린더 + 다이어리 동시 입력**: 메모에 날짜·시간·회의·행사·마감 등
   일정 정보가 있으면 두 곳에 동시에 넣는다:
   - Google Calendar MCP `create_event` — 제목·일시(시간 없으면 종일), 설명에 원본 Inbox 링크
   - 📓 다이어리 항목 — `날짜`=해당 일정일, `선택`=업무/행사 등, `관련 자료`=원본,
     `설명`에 "📅 구글 캘린더 등록됨" 표기
   - 날짜가 모호하면("다음 주쯤") 임의로 정하지 말고 사용자에게 확인 후 등록
   - 캘린더 등록 전 [제목·일시] 목록을 보여주고 진행 (외부 캘린더는 되돌리기 번거로움)
6. **보고**: [인사이트 제목 → 요약 한 줄 → 생성한 액션] 목록으로 정리해 보여준다. 5건 초과면 표로.

## 규칙

- 원문 삭제·수정 금지 — 요약은 `요약` 필드에만.
- 학생 실명·개인정보가 요약에 그대로 노출되지 않게 일반화("학생 A").
- 항목이 20건 넘으면 먼저 개수를 알리고 계속할지 확인.
- 앱에서 아직 CSV로 안 옮긴 메모가 있다고 하면: 곰곰 비서 → "노션용 CSV" → ✅ Inbox DB에서 가져오기 안내.
