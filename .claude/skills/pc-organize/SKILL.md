# PC 폴더·스크린샷 정리 스킬

PC(로컬 세션)에서 흩어진 파일을 정리하고, 가치 있는 스크린샷을 **인사이트 파이프라인**에 태워 보낸다.

## 연결 구조 (이 스킬의 위치)

```
PC 폴더/스크린샷  ──[pc-organize]──▶  정리된 폴더
        │
        ├─ 쓸 만한 스크린샷 → Google Drive 업로드 폴더 ──[drive-images]──▶ 자료 제작(PPT 등)
        └─ 아이디어·기록거리 → 노션 "💡 인사이트 정리" DB의 inbox
                                  (데이터소스: collection://ec1dff42-c7ff-4373-8801-49d5894f3c74)
                                  대시보드 Inbox(빠른 수집함)와 연결되어 있음
```

## 정리 절차

1. **스캔**: 대상 폴더(바탕화면·다운로드·스크린샷 폴더)의 파일 목록을 확장자·날짜·크기로 파악. 삭제는 절대 먼저 하지 않는다.
2. **분류 규칙 제안**: 파일명·시간대 패턴으로 묶음을 제안하고 사용자 승인 후 이동.
   - 스크린샷: `스크린샷 YYYY-MM-DD HHMMSS` 패턴 → 날짜별 폴더 `스크린샷/YYYY-MM/`
   - 수업 관련(수업 시간대·수업 도구 화면) → `수업자료/과목-단원/`
   - 문서·PDF·PPT → `문서/주제/`
3. **이름 변경**: 내용을 Read로 확인해 `YYYY-MM-DD-간단설명.png` 형태로 변경 (원본명은 메모로 보존).
4. **선별 → 파이프라인 전송**:
   - 자료 제작에 쓸 화면 → Drive 업로드 폴더에 복사 (사용자가 동기화 폴더 지정, 예: `내 드라이브/스크린샷/`) → 이후 `drive-images` 스킬이 수집
   - 아이디어·기록 → 노션 MCP `notion-create-pages` 로 "💡 인사이트 정리" DB에 상태=`inbox`, 출처=`스크린샷` 항목 생성 (첨부는 Drive 링크로)
5. **삭제는 마지막에, 승인 후에만**: 중복(해시 동일)·빈 파일만 후보로 제시하고, 지우기 전 목록을 보여준다.

## 안전 규칙

- **원본 우선**: 이동은 언제나 복사→확인→정리 순서. 덮어쓰기 금지(이름 충돌 시 `-2` 붙임).
- **개인정보**: 계정·주민번호·학생 실명이 보이는 파일은 별도 `민감/` 폴더로 격리하고 사용자에게 알림.
- **일괄 작업 전 미리보기**: 옮길 파일 목록과 대상 경로를 표로 보여주고 승인받는다.
- PC 로컬 파일은 클라우드 세션에서 직접 접근 불가 — 이 스킬은 **사용자 PC에서 클로드 코드를 실행 중일 때** 적용된다. 클라우드 세션이라면 Drive 업로드 후 `drive-images` 경로를 안내한다.

## 자주 쓰는 명령 예

- "다운로드 폴더에서 스크린샷만 골라서 날짜별로 정리해줘"
- "바탕화면 정리하고, 수업 관련 캡쳐는 드라이브 스크린샷 폴더로 복사해줘"
- "이 폴더에서 쓸 만한 화면 골라서 인사이트 inbox에 넣어줘"
