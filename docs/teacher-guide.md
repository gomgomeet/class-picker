# 선생님을 위한 클로드 코드 활용 가이드

> 코딩을 몰라도 괜찮아요. 말로 시키면 AI가 만들어주는 시대, 학급 도구를 직접 만들어 쓰는 방법을 정리했습니다.
> 실제 사례: 수강신청/자리뽑기 웹앱 `class-picker` (https://github.com/gomgomeet/class-picker)

---

## 1. 제일 중요한 원리: 작업이 사라지지 않게 하는 법

**대화와 파일은 저장되는 곳이 달라요.**

| 무엇 | 어디에 저장? | 다른 컴퓨터에서 보여요? |
|---|---|---|
| Claude와의 대화 | 그 컴퓨터 안 | ❌ 안 보여요 |
| 작업 파일 (커밋&푸시 전) | 그 컴퓨터 안 | ❌ 안 보여요 |
| 작업 파일 (커밋&푸시 후) | GitHub (온라인) | ✅ 어디서든 |

그래서 규칙은 하나예요: **작업이 끝나면 반드시 "저장해줘"라고 말하기.**
그러면 Claude가 ① 오늘 한 일을 작업 일지(PROGRESS.md)에 기록하고 ② 커밋(사진 찍기) ③ 푸시(온라인 업로드)까지 해줘요.

깜빡해도 괜찮아요 — 세션이 끝날 때 자동으로 백업하는 안전망(hook)도 설정해뒀어요.

## 2. 에이전트가 뭐예요?

**에이전트 = 한 가지 일을 잘하는 AI 조수**예요. 각 조수에게 "언제 나서야 하는지"와 "일하는 방법"을 적은 설명서(md 파일 하나!)를 만들어주면 돼요.

이름을 외울 필요 없이 평소처럼 말하면, Claude가 알아서 맞는 조수를 불러요.

## 3. 우리 반 에이전트 사용 설명서

| 이렇게 말하면 | 누가 나서요 | 하는 일 |
|---|---|---|
| "저장해줘" / "오늘 끝" | save-work | 작업 일지 기록 + 커밋 + 푸시 |
| "○○ 기능 만들고 싶어" | feature-planner | 질문 후 PRD(계획서) 작성 |
| "알아서 만들어줘" | auto-build (총감독) | 계획→제작→검사→수리→저장 자동 진행 |
| (에러 붙여넣고) "왜 안 돼?" | error-doctor | 에러를 쉬운 말로 진단하고 수리 |
| "버그 찾아줘" | bug-hunter | 버그를 찾아 보고, 작은 건 직접 수리 |
| "잘 되는지 테스트해줘" | app-tester | 실제 브라우저로 열어 검사 + 스크린샷 |
| "왜 이렇게 느려?" | perf-doctor | 느린 원인 진단 보고서 |
| "더 보기 좋게 해줘" | ui-designer | 학생 눈높이로 화면 다듬기 |
| "이 코드 뭐 하는 거야?" | code-explainer | 초보자 눈높이 설명 (수정 안 함) |
| "튜토리얼 만들어줘" | tutorial-writer | 튜토리얼/발표자료 초안 작성 |
| "어제 버전으로 되돌려줘" | time-machine | git 기록에서 예전 버전 복구 |
| "개인정보 확인해줘" | privacy-checker | 학생 실명 등 노출 위험 점검 |
| "배포해줘" | deploy-helper | 학생들이 접속할 웹 주소 만들기 |
| "허용 창 그만 뜨게 해줘" / "전역으로 설정해줘" | permission-setup | 안전한 명령 허용 목록 설정 |
| "노션에 정리해줘" | notion-sync | 작업 기록/계획서를 노션으로 |

## 4. 추천 작업 흐름

```
계획 → 제작 → 테스트 → 수리 → 개인정보 점검 → 저장(=자동 배포)
```

또는 그냥: **"○○ 알아서 만들어줘"** — 총감독(auto-build)이 위 공정을 순서대로 지휘해요. 계획서(PRD) 승인 때 딱 한 번만 확인하고, 나머지는 자동이에요.

## 5. 새 컴퓨터 설정법 (학교 ↔ 집)

딱 세 줄이면 어느 컴퓨터든 같은 환경이 돼요:

```powershell
# 1. 프로젝트 받기
git clone https://github.com/gomgomeet/class-picker.git

# 2. 에이전트 모음을 개인 폴더에 설치 (모든 프로젝트에서 사용 가능)
git clone https://github.com/gomgomeet/my-claude-agents $HOME\.claude\agents

# 3. 이후 업데이트는 각 폴더에서
git pull
```

- 에이전트 모음 저장소: https://github.com/gomgomeet/my-claude-agents
- 노션 연동을 쓰려면: claude.ai 설정 → 커넥터에서 Notion 연결 (컴퓨터마다 한 번)

## 6. 허용 확인 창 줄이기

"허용하시겠습니까?" 창은 AI를 감독하는 안전장치라 AI가 스스로 통과할 수 없어요. 대신:

- 아무 프로젝트에서나 **"권한 전역으로 설정해줘"** → 안전한 읽기 명령들은 확인 없이 실행돼요
- 창이 떴을 때 **"항상 허용"**을 고르면 그 명령은 다음부터 안 물어봐요
- 파일 삭제 같은 위험한 명령은 계속 물어봐요 — 일부러 남겨둔 안전장치예요

## 7. 자주 묻는 질문

**Q. 학교에서 한 대화가 집에서 안 보여요.**
A. 정상이에요. 대화는 그 컴퓨터에만 저장돼요. 그래서 작업 내용을 파일로 남겨 푸시하는 습관이 중요해요. 이어서 하려면 새 대화에서 "PROGRESS.md 읽고 이어서 하자"라고 하면 돼요.

**Q. 코드를 잘못 고쳐서 망가뜨렸어요.**
A. "어제 버전으로 되돌려줘"라고 하세요. 커밋해둔 모든 버전은 git에 남아 있어요.

**Q. 학생들이 쓰게 하려면?**
A. "배포해줘" → GitHub Pages 주소가 생겨요. 이후에는 저장(푸시)할 때마다 자동으로 최신 버전이 반영돼요. 배포 전에 "개인정보 확인해줘"로 학생 실명이 없는지 꼭 점검하세요.
