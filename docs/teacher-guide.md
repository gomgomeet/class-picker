# 선생님을 위한 클로드 코드 자동화 가이드

class-picker 프로젝트에서 만든 자동화 도구들을 처음 쓰는 분 눈높이로 정리한 자료입니다.
(Notion "📘 클로드 작업 노트"에도 같은 내용이 카드 4개로 정리되어 있습니다.)

---

## 1. 클로드가 매번 물어보는 확인 창, 안전하게 줄이기 (난이도: 쉬움)

### 이 확인 창은 뭔가요?

클로드 코드는 터미널 명령을 실행하기 전에 **"Do you want to proceed?"** 하고 물어봅니다. 파일을 지우는 등 위험한 일을 막아주는 **안전벨트**예요. 좋은 기능이지만, 안전한 명령까지 매번 물어보면 번거롭죠.

> 💡 왜 유독 많이 뜰까? 리눅스/맥 명령어(`ls`, `cat` 등)는 "읽기만 하는 안전한 명령"으로 자동 인식되지만, **Windows PowerShell 명령어는 자동 인식이 안 돼서** 매번 물어봅니다.

### 방법 1. 그 자리에서 2번 선택 (제일 간단)

확인 창에서 **2번 "Yes, and don't ask again for..."**를 선택하면 같은 명령은 다시 묻지 않아요. 복잡한 반복문 명령은 이 방법이 제일 확실합니다.

### 방법 2. 허용 목록 등록 (추천)

프로젝트 폴더의 `.claude/settings.json` 파일에 안전한 명령을 미리 등록해 둡니다.

```json
{
  "permissions": {
    "allow": [
      "Bash(Get-ChildItem *)",
      "Bash(Get-Content *)",
      "Bash(Select-String *)",
      "Bash(Test-Path *)"
    ]
  }
}
```

- 직접 편집할 필요 없이 클로드에게 **"허용 목록 설정해줘"**라고 말하면 됩니다 (permission-setup 에이전트가 대신 해줌)
- 이 파일은 GitHub에 함께 올라가므로, 같은 프로젝트를 받는 **모든 컴퓨터에 자동 적용**됩니다
- 단, **그 프로젝트 폴더에서만** 적용돼요. 모든 프로젝트에 적용하려면 클로드 코드에서 `/permissions` → **User settings**에 등록하세요

### 방법 3. 확인을 아예 끄기 (비추천 ⚠️)

`claude --dangerously-skip-permissions` 명령으로 끌 수 있지만, 이름 그대로 위험해서 초보 단계에서는 권하지 않습니다.

### 주의: 허용하면 안 되는 것

아래 명령은 아무 코드나 실행하거나 파일을 지울 수 있어서 허용 목록에 넣으면 안 돼요. 방법 1로 그때그때 허용하는 게 안전합니다.

- `Remove-Item`(삭제), `python`, `node` 같은 인터프리터 전체 허용
- `Invoke-Expression`, `ForEach-Object` (아무 코드나 실행 가능)
- `Invoke-RestMethod` 전체 허용 (아무 인터넷 주소로나 요청 가능)

## 2. 에이전트: 나만의 AI 전문가 직원 만들기 (난이도: 보통)

### 에이전트가 뭔가요?

에이전트는 **역할과 일하는 방법을 적어둔 md 파일 하나**입니다. 클로드에게 "버그 찾아줘"라고 말하면, 클로드가 해당 전문 에이전트를 불러서 일을 시켜요. **전문 분야가 정해진 직원을 고용하는 것**과 같습니다.

### 우리가 만든 에이전트 5종

| 에이전트 | 하는 일 | 부르는 말 |
|---|---|---|
| permission-setup | 확인 창 줄이는 허용 목록을 자동 설정 | "허용 목록 설정해줘" |
| perf-doctor | 웹앱이 느린 원인을 진단, 처방전 보고서 작성 | "왜 이렇게 느려?" |
| bug-hunter | 버그를 찾아 보고하고 작은 버그는 직접 수정+검증 | "버그 찾아줘" |
| tutorial-writer | 코드를 분석해 초보자용 튜토리얼/발표자료 작성 | "튜토리얼 만들어줘" |
| code-explainer | 코드를 초보자 눈높이로 설명 (수정 안 함) | "이 코드 뭐 하는 거야?" |

### 에이전트 파일은 이렇게 생겼어요

`.claude/agents/` 폴더에 md 파일로 저장합니다.

```markdown
---
name: bug-hunter
description: 버그를 찾아 보고하고 고쳐주는 에이전트. "버그 찾아줘"라고 하면 사용한다.
tools: Read, Glob, Grep, Bash, Edit
---

너는 버그 사냥꾼이다. 버그를 찾아 한국어로 보고하고, 확실한 것만 고친다.
(...일하는 순서와 원칙을 적는다...)
```

- 맨 위 `---` 사이: 이름, 언제 쓰는지, 쓸 수 있는 도구 목록
- 그 아래: 에이전트에게 주는 업무 지시서 (한국어로 써도 됩니다!)

### 만들 때 팁 (안전장치 넣기)

- **진단 전문이면 수정 도구를 빼기**: perf-doctor는 tools에 `Edit`가 없어서 코드를 절대 못 고칩니다. 의사가 진단서만 쓰고 수술은 확인 후 하는 구조
- **금지 목록 넣기**: permission-setup 안에는 "위험한 명령은 사용자가 요청해도 추가하지 말 것"이라는 금지 목록이 있어요
- **보고서 형식 정해주기**: "한 줄 요약 → 원인 → 조치 → 검증" 형식을 지시서에 넣으면 결과물이 일정해집니다

### 직접 만들려면

클로드에게 **"○○하는 에이전트 만들어줘"**라고 말하면 끝. 파일 형식을 몰라도 클로드가 알아서 만들어 줍니다. 만든 뒤 `/agents` 명령으로 목록을 확인할 수 있어요.

## 3. 에이전트를 모든 컴퓨터·모든 프로젝트에서 쓰기 (난이도: 보통)

### 문제: 에이전트는 만든 곳에서만 동작해요

- 프로젝트 폴더 안의 `.claude/agents/` → **그 프로젝트에서만** 동작
- 컴퓨터의 `~/.claude/agents/` (전역 폴더) → **그 컴퓨터의 모든 프로젝트**에서 동작하지만, **컴퓨터마다 따로** 설치해야 함

### 해결: 전역 폴더 자체를 GitHub 저장소로!

에이전트 전용 저장소(예: `my-claude-agents`)를 만들고, **전역 폴더 위치에 바로 clone**합니다.

```powershell
git clone https://github.com/내아이디/my-claude-agents $HOME\.claude\agents
```

이 한 줄이면 새 컴퓨터 설치 끝! 폴더 자체가 git 저장소라서 **복사 단계가 아예 사라집니다.**

### 일상 사용법

최신 버전 받기 (다른 컴퓨터에서 수정했을 때):

```powershell
cd $HOME\.claude\agents
git pull
```

에이전트를 새로 만들거나 고쳤을 때:

```powershell
cd $HOME\.claude\agents
git add -A
git commit -m "새 에이전트 추가"
git push
```

### 만드는 순서 (한 번만)

1. GitHub에서 새 저장소 만들기 (https://github.com/new, **Private** 추천)
2. 클로드에게 "에이전트들을 새 저장소로 옮겨줘"라고 부탁 → 에이전트 파일 + 사용법 README를 넣어줌
3. 각 컴퓨터에서 위의 clone 한 줄 실행

> 💡 원리: `~/.claude/agents`는 클로드 코드가 "전역 에이전트"를 찾는 약속된 위치예요. 그 위치에 git 저장소를 두면 "동기화되는 전역 폴더"가 됩니다. 대화 기록은 컴퓨터마다 따로지만, **파일은 GitHub로 어디서든 이어집니다.**

## 4. 작업 자동 백업: /save-work 스킬과 안전망 훅 (난이도: 보통)

### 왜 필요한가요?

클로드와의 **대화 기록은 컴퓨터마다 따로** 저장됩니다. 집에서 하던 작업을 학교에서 이어가려면, 작업 내용이 **파일로 남아서 GitHub에 올라가 있어야** 해요. 그런데 매번 "기록하고, 커밋하고, 푸시"를 손으로 하면 잊어버리기 쉽죠.

### 스킬(Skill)이란?

**자주 하는 일의 순서를 적어둔 md 파일**입니다. 에이전트가 "직원"이라면, 스킬은 "업무 매뉴얼"이에요. `.claude/skills/스킬이름/SKILL.md`에 저장하면 `/스킬이름`으로 실행할 수 있습니다.

### /save-work 스킬이 하는 일

사용자가 **"저장해줘", "백업해줘", "오늘 끝"**이라고 말하면:

1. **PROGRESS.md 기록** — 오늘 한 일 / 변경된 파일 / **다음에 할 일**을 작업 일지 맨 위에 추가
2. **커밋** — 실제 작업 내용을 요약한 한국어 커밋 메시지로
3. **푸시** — GitHub에 백업 (실패하면 자동 재시도)

다음 날 다른 컴퓨터에서 클로드를 켜면, PROGRESS.md의 "다음에 할 일"을 읽고 **어제 하던 작업을 그대로 이어갑니다.**

### 안전망: 세션 종료 훅(Hook)

저장을 깜빡하고 창을 닫아도 괜찮도록, **세션이 끝날 때 자동으로 커밋&푸시하는 훅**을 걸어뒀습니다. `.claude/settings.json`에 등록:

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          { "type": "command", "command": "bash .claude/hooks/auto-backup.sh" }
        ]
      }
    ]
  }
}
```

> 💡 훅(hook)은 "특정 순간에 자동으로 실행되는 명령"이에요. 스킬은 말로 시켜야 돌지만, 훅은 **말 안 해도** 정해진 순간에 돕니다.

### 규칙까지 넣으면 완성 (CLAUDE.md)

프로젝트의 `CLAUDE.md`에 규칙을 적어두면 클로드가 항상 지킵니다:

- "중요한 작업이 끝날 때마다 /save-work를 실행한다"
- "새 세션을 시작하면 먼저 PROGRESS.md를 읽고 이어서 작업한다"

이 세 가지(스킬 + 훅 + CLAUDE.md 규칙)를 합치면, **어느 컴퓨터에서든 끊기지 않는 작업 흐름**이 만들어집니다.
