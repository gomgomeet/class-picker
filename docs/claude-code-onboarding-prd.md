# PRD: 클로드 코드(Claude Code) 온보딩 PPT & 튜토리얼 영상

| 항목 | 내용 |
|---|---|

| 작성일 | 2026-07-10 |
| 상태 | Draft (검토 대기) |
| 산출물 | ① 온보딩 PPT ② 튜토리얼 영상 시리즈 |

---

## 1. 배경 및 문제 정의

클로드 코드는 터미널 기반 AI 코딩 에이전트로, 단순 코드 자동완성을 넘어 코드베이스 이해, 멀티파일 수정, Git 연동, 외부 도구 연결(MCP) 등 기능이 매우 많다. 그러나 처음 사용하는 사람 입장에서는:

- **기능이 너무 많아** 무엇부터 배워야 할지 모른다.
- 터미널(CLI) 환경이 낯설어 **진입 장벽**을 느낀다.
- 슬래시 커맨드, CLAUDE.md, MCP, 훅(Hooks) 등 **용어가 생소**하다.
- 공식 문서는 영어 중심이고 레퍼런스 형태라 **학습 순서가 없다**.

→ "처음 켜서 → 실제 업무에 쓰기까지"를 단계별로 안내하는 **한국어 온보딩 자료**가 필요하다.

## 2. 목표 (Goals)

1. 클로드 코드를 처음 접하는 사람이 **30분 안에 설치부터 첫 작업 완료**까지 도달하게 한다.
2. 핵심 기능을 **기초 → 중급 → 고급** 3단계 커리큘럼으로 구조화하여, 자기 수준에 맞게 골라 배울 수 있게 한다.
3. PPT(발표/공유용)와 영상(자습용)이 **동일한 커리큘럼**을 공유하여, 어느 쪽으로 배워도 같은 내용을 얻게 한다.

### 목표가 아닌 것 (Non-Goals)

- Claude API / Agent SDK 개발 가이드 (별도 자료로 분리)
- 프롬프트 엔지니어링 심화 강의
- 특정 프로그래밍 언어 강의

## 3. 대상 사용자 (Target Audience)

| 페르소나 | 설명 | 니즈 |
|---|---|---|
| **P1. 입문 개발자** | 터미널 사용 경험이 적은 주니어 개발자, 학생 | 설치·기본 사용법을 손잡고 알려주는 자료 |
| **P2. 현업 개발자** | IDE 중심으로 일하며 AI 도구를 처음 도입 | 실무 워크플로(Git, 리뷰, 테스트)에 어떻게 붙이는지 |
| **P3. 비개발 직군** | 기획자·디자이너 등, 웹/앱(claude.ai, Cowork)에서 사용 | 코딩 없이도 쓸 수 있는 시나리오 |

주 타깃은 P1·P2이며, PPT 도입부에 P3용 안내(웹 버전 소개)를 짧게 포함한다.

## 4. 다뤄야 할 기능 범위 (커리큘럼 맵)

### Level 1 — 시작하기 (기초)

| 기능 | 핵심 메시지 |
|---|---|
| 설치 & 로그인 | `npm install -g @anthropic-ai/claude-code` → 프로젝트 폴더에서 `claude` 실행 |
| 기본 대화 | 자연어로 질문·요청하면 Claude가 파일을 읽고, 수정하고, 명령을 실행함 |
| 코드베이스 질문 | "이 프로젝트 구조 설명해줘", "로그인 로직 어디 있어?" |
| 파일 수정 요청 | 수정 전 diff를 보여주고 승인받는 흐름 (권한 시스템) |
| 필수 슬래시 커맨드 | `/help`, `/clear`, `/model`, `/init`, `/config` |
| 대화 이어가기 | `claude -c` (이전 대화 재개), `Esc`로 중단 |

### Level 2 — 업무에 붙이기 (중급)

| 기능 | 핵심 메시지 |
|---|---|
| CLAUDE.md 메모리 | 프로젝트 규칙·컨벤션을 기억시키기, `/init`으로 자동 생성, `#` 단축 입력 |
| Plan Mode | `Shift+Tab`으로 전환 — 실행 전 계획을 먼저 검토 |
| 권한 모드 | 매번 승인 / 자동 승인(auto-accept) 등 모드 이해와 안전한 사용 |
| Git 워크플로 | 커밋 메시지 작성, 브랜치 작업, PR 생성, 머지 컨플릭트 해결 |
| 이미지 활용 | 스크린샷·디자인 시안을 붙여넣어 UI 구현/버그 리포트 |
| 확장 사고 | 어려운 문제에 "think hard" 등으로 더 깊은 추론 유도 |
| 컨텍스트 관리 | `/compact`, `/clear` — 긴 세션에서 대화 정리하기 |
| 체크포인트/되돌리기 | `Esc Esc`(rewind)로 이전 상태로 복원 |

### Level 3 — 확장하기 (고급)

| 기능 | 핵심 메시지 |
|---|---|
| 커스텀 슬래시 커맨드 & 스킬 | 반복 작업을 `.claude/commands`, 스킬로 자동화 |
| 서브에이전트 (Subagents) | 역할별 에이전트(리뷰어, 테스터 등) 정의·병렬 실행 |
| MCP 연결 | Slack, Notion, DB 등 외부 도구를 Claude에 연결 |
| Hooks | 툴 실행 전후 자동 동작(린트, 알림 등) 걸기 |
| IDE 통합 | VS Code / JetBrains 확장으로 에디터 안에서 사용 |
| 웹 & 모바일 | claude.ai/code — 클라우드 세션, 이동 중 작업 확인 |
| GitHub Actions | PR에서 `@claude` 멘션으로 자동 리뷰·수정 |
| Headless 모드 | `claude -p "..."` 로 스크립트/CI에서 활용 |

## 5. 산출물 ① — 온보딩 PPT

### 5.1 요구사항

- **분량**: 35~45장 (발표 시간 40~50분 기준)
- **언어**: 한국어 (명령어·용어는 영어 병기)
- **형식**: 16:9, 발표용 + 배포용(노트 포함) 두 버전
- **디자인**: 다크 테마 터미널 스크린샷 중심, 슬라이드당 메시지 1개 원칙

### 5.2 목차 구성 (안)

| 섹션 | 슬라이드 수 | 내용 |
|---|---|---|
| 1. 인트로 | 3~4 | 클로드 코드란? 무엇이 다른가 (에이전트 vs 자동완성), 데모 티저 |
| 2. 설치 & 첫 실행 | 4~5 | 설치, 로그인, 첫 대화, 권한 승인 흐름 |
| 3. Level 1 핵심 기능 | 7~8 | 코드베이스 질문, 파일 수정, 슬래시 커맨드 (라이브 데모 1) |
| 4. Level 2 실무 워크플로 | 8~10 | CLAUDE.md, Plan Mode, Git/PR, 이미지 활용 (라이브 데모 2) |
| 5. Level 3 확장 기능 | 6~8 | MCP, 서브에이전트, 훅, IDE/웹 — "이런 것도 된다" 수준의 소개 |
| 6. 실전 팁 & 안티패턴 | 4~5 | 좋은 요청 vs 나쁜 요청, 컨텍스트 관리, 비용 관리 |
| 7. 마무리 | 2~3 | 학습 로드맵, 공식 문서/커뮤니티 링크, Q&A |

### 5.3 슬라이드 작성 원칙

- 모든 기능 슬라이드는 **[한 줄 정의 → 실행 화면 → 언제 쓰는지]** 3요소로 통일
- 명령어는 복사 가능한 텍스트로도 부록에 정리 (치트시트 슬라이드 1장 포함)
- 데모 실패 대비: 라이브 데모 구간마다 백업 스크린샷/GIF 준비

## 6. 산출물 ② — 튜토리얼 영상

### 6.1 요구사항

- **형식**: 시리즈형 짧은 영상 (편당 5~10분), 총 6~8편
- **구성**: 화면 녹화(터미널) + 자막 + 내레이션(한국어)
- **해상도**: 1080p 이상, 터미널 폰트 크게 (모바일 시청 고려)
- **편집**: 타이핑 대기 구간 배속 처리, 핵심 명령어는 화면에 자막 오버레이

### 6.2 에피소드 구성 (안)

| 편 | 제목 | 길이 | 내용 |
|---|---|---|---|
| EP1 | 클로드 코드 5분 만에 시작하기 | 5분 | 설치 → 로그인 → 첫 질문 → 첫 수정 |
| EP2 | 내 프로젝트 이해시키기 | 7분 | 코드베이스 탐색, `/init`, CLAUDE.md |
| EP3 | 안전하게 코드 수정하기 | 7분 | 권한 시스템, Plan Mode, 되돌리기 |
| EP4 | Git 작업 맡기기 | 8분 | 커밋, 브랜치, PR 생성까지 실전 흐름 |
| EP5 | 실전 프로젝트: 기능 하나 만들기 | 10분 | EP1~4 종합 — 작은 기능을 처음부터 끝까지 |
| EP6 | 반복 작업 자동화 | 8분 | 커스텀 커맨드, 스킬, 훅 |
| EP7 | 외부 도구 연결 (MCP) | 8분 | MCP 개념, 대표 서버 연결 실습 |
| EP8 | 어디서든 쓰기 | 6분 | IDE 확장, 웹(claude.ai/code), GitHub Actions |

- EP1~EP5가 **필수 코스**, EP6~EP8은 선택 코스로 안내
- 각 영상 끝에 "다음 편 예고 + 직접 해볼 과제 1개" 포함

### 6.3 영상별 공통 포맷

1. (10초) 이번 편에서 배울 것 미리보기
2. (본문) 화면 녹화 + 내레이션 — 실제 프로젝트 기준 시연
3. (30초) 요약 + 따라 해볼 과제 + 다음 편 안내

## 7. 데모/실습용 예제 프로젝트

- PPT와 영상 모두 **동일한 예제 프로젝트**를 사용해 일관성 유지
- 조건: 언어 장벽이 낮은 소규모 웹 프로젝트(예: 간단한 HTML/JS 앱 — 본 저장소 class-picker 같은 규모), 일부러 남겨둔 작은 버그 2~3개 포함(수정 데모용)
- 예제 저장소는 공개 GitHub repo로 배포하여 시청자가 그대로 따라 할 수 있게 함
