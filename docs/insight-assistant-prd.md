# PRD: 인사이트 정리 비서 (가제)

> 흩어진 수업·업무 자료를 한 곳에 모아 자동으로 정리하고, 수업자료(PPT·동영상·차트)와 평가/연구로 연결해 주는 핸드폰 중심 개인 비서 앱

|---|---|

| 작성일 | 2026-07-14 |

## 1. 배경 · 문제

교사(사용자)는 하루에도 여러 경로로 자료가 쏟아진다 — 수업 사진, 스크린샷, 음성 메모, 메시지·톡, 다운로드 파일, 블로그·인스타·쓰레드에서 본 것들. 그런데:

- 자료가 **갤러리·메신저·다운로드 폴더에 흩어져** 나중에 못 찾는다.
- 수업자료(PPT·동영상·차트)로 만들려면 **매번 손으로 다시 모아야** 한다.
- 수업 활동 피드백을 **평가·연구(통계)로 연결**하는 흐름이 없다.

→ 입력을 한 번에 담고(inbox), 자동으로 분류하고(인사이트/Diary/할일), 산출물로 뽑아내는 **개인 비서**가 필요하다.

## 2. 목표

1. **모으기 마찰 0** — 핸드폰에서 녹음/사진/공유 한 번이면 자료가 inbox에 들어온다.
2. **자동 정리** — inbox의 항목을 인사이트·다이어리·할일로 자동 분류한다.
3. **산출물 연결** — 모인 인사이트에서 PPT·동영상·차트를 만들고, 수업 평가/연구(t·f검정)로 잇는다.

### 목표가 아닌 것 (Non-Goals, 최소한 초기엔)

- 완전 자동 AI 분류의 100% 정확도 (사람이 확인·수정하는 반자동으로 시작)
- 타인과의 협업·공유 플랫폼 (개인용 우선)
- 갤러리 완전 자동 감시 (핸드폰 OS 제약 — 3차 과제)

## 3. 대상 사용자

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| **P1. 교사 본인** | 수업·업무 자료가 많고 정리 시간이 부족 | 빠른 수집 + 자동 정리 + 수업자료 생성 |
| P2. 연구하는 교사 | 수업 활동 데이터를 평가·연구로 | 인사이트→통계(t·f검정)→평가 방법 |

## 4. 핵심 사용자 시나리오

1. **음성 메모** — 녹음 버튼 → (필요시 번역) → inbox에 텍스트로 저장
2. **화면 캡쳐** — 스크린샷을 공유 시트로 앱에 보냄 → "업무자료/다이어리"로 태그
3. **SNS 갈무리** — 블로그·인스타·쓰레드에서 공유 → 인사이트로 수집
4. **정리 시간** — inbox를 훑으며 인사이트 / Diary(할일) / 삭제로 분류
5. **산출물** — 특정 주제의 인사이트를 골라 → PPT 초안 / 차트(설문·점수) / 동영상 스크립트 생성
6. **평가·연구** — 수업 활동 점수를 모아 t검정·f검정 → 평가 방법 리포트

## 5. 기능 명세 (모듈별 · 우선순위)

### 모듈 A. 빠른 수집 (Capture) — MVP
- 텍스트/음성 메모 입력, 음성→텍스트 변환(+번역 옵션)
- 사진·스크린샷 업로드(공유 시트 or 파일 선택)
- 링크(블로그·인스타·쓰레드 URL) 붙여넣기 → 제목·요약 자동 추출
- 모든 입력은 일단 **inbox**로

### 모듈 B. 자동 분류 (Organize) — MVP
- inbox 항목을 **인사이트 / Diary(할일) / 업무자료 / 삭제**로 분류
- 스크린샷 vs 일반 사진 구분(해상도·메타데이터 힌트)
- 규칙 + 사용자가 고치는 반자동. 자주 쓰는 태그 학습

### 모듈 C. 인사이트 허브 (Insight DB) — MVP 핵심
- 출처 태그: 메시지·톡 / 다운로드 / 스크린샷 / 쓰레드 / 블로그 / 인스타
- 주제 태그, 날짜, 원본 링크, 메모, 첨부(이미지·파일)
- 검색·필터·보드 뷰 (아래 6장 스키마)

### 모듈 D. 산출물 생성 (Produce) — 2차
- 선택한 인사이트 묶음 → **PPT 초안** 자동 생성 (기존 class-deliverables 파이프라인 재사용)
- **차트**: 표/설문 데이터 → 막대·꺾은선; 통계 **t검정·f검정** 자동 계산
- **동영상 스크립트/자막** 초안

### 모듈 E. 수업 평가·연구 (Research) — 2차
- 수업 활동별 피드백·점수 누적
- 사전/사후, 집단 간 비교 → t·f검정 → 평가 방법 리포트

### 모듈 F. 개인 비서 부가기능 — 3차
- 할일(Diary), 건강관리(운동·식사·물 마시기 체크), 책기록
- 갤러리 기본 폴더 자동 인식(네이티브 앱 필요)

## 6. 데이터 구조 — 인사이트 DB 스키마(안)

| 필드 | 타입 | 설명 |
|---|---|---|
| 제목 | text | 한 줄 제목 |
| 상태 | select | inbox · 인사이트 · Diary · 보관 |
| 출처 | multi-select | 메시지·톡 / 다운로드 / 스크린샷 / 쓰레드 / 블로그 / 인스타 / 음성 |
| 주제 | multi-select | 수업/평가/연구/개인 등 자유 태그 |
| 원본링크 | url | SNS·파일 원본 |
| 첨부 | files | 이미지·PDF |
| 메모 | text | 요약·생각 |
| 날짜 | date | 수집일 |
| 산출물 | multi-select | PPT / 동영상 / 차트 로 만들었는지 |

> 이 스키마는 노션 DB로 바로 만들 수 있고, 웹앱으로 갈 때도 그대로 테이블 설계로 쓸 수 있음.

## 7. 기술 방향 (현실적 단계)

- **1단계 (가장 쉬움)**: 노션 DB + 노션 공유 시트로 수집 → 익숙한 도구로 인사이트 허브부터 시작
- **2단계**: 간단한 **웹앱**(핸드폰 브라우저에서 열림). 업로드·태그·검색 + PPT/차트 생성은 기존 `class-deliverables` 스크립트 재사용
- **3단계**: 네이티브 앱(갤러리 자동 인식, 공유 시트 확장, 백그라운드 수집)
- 통계(t·f검정)는 파이썬(scipy) 또는 웹에서 계산

## 8. 개인정보 · 안전

- 학생 사진·이름·성적이 들어올 수 있음 → **개인 기기·개인 계정에만 저장**, 공개 배포 금지
- 산출물(PPT·차트) 공유 전 개인정보 마스킹 체크
- SNS 원본 링크만 저장, 저작권 있는 콘텐츠는 참고용

## 9. 성공 지표

- 자료 수집까지 걸리는 시간: **10초 이내**(녹음·공유 한 번)
- inbox → 정리 완료 비율: 주 1회 정리로 80% 이상
- 수업자료 1건 만드는 시간: 기존 대비 절반

## 10. 마일스톤(안)

| 단계 | 산출물 |
|---|---|
| M1 | 인사이트 DB 스키마 확정 + 노션으로 수집 시작 |
| M2 | 웹앱 MVP: 업로드·태그·검색(핸드폰에서 작동) |
| M3 | 인사이트 → PPT·차트 생성 연결 |
| M4 | 수업 평가·연구(t·f검정) 모듈 |
| M5 | 개인 비서 부가기능 + (선택) 네이티브 |

## 10.5 요약 기능 — 결정 기록과 업그레이드 루트 (2026-07 확정)

**현재(②안 채택)**: 음성 메모는 앱→CSV→노션 ✅ Inbox로 옮긴 뒤, `inbox-summarize` 스킬로
클로드에게 "인박스 요약해줘" 하면 요약 필드가 채워진다. 추가 비용 없음.

**①안(앱 내 실시간 요약)으로 전환하는 루트** — 마음에 안 들면 이대로 구현:

1. Anthropic API 키 발급 (console.anthropic.com, 종량제) →
   Vercel 프로젝트 → Settings → Environment Variables → `ANTHROPIC_API_KEY` 등록
2. 저장소에 `api/summarize.js` 서버리스 함수 추가:
   - POST {text} 받아 Anthropic Messages API 호출 (모델: claude-haiku 계열, max_tokens≈300)
   - 프롬프트: "다음 음성 전사를 2~3문장으로 요약하고, 인사이트/할일 중 무엇인지 분류" → JSON 응답
   - CORS는 같은 도메인이라 불필요, 키는 서버에만 존재
3. `gomgom/index.html` 담기 칸에 **✨ 요약 버튼** 추가:
   - fetch('/api/summarize') → 결과로 입력창 교체 + 분류 자동 선택
   - 함수가 404/키 미설정이면 버튼 자동 숨김 (②안으로 자연 폴백)
4. 확장: 같은 함수로 "자동 분류"(inbox 항목 일괄 정리) 버튼도 가능
- 예상 작업량: 1세션(1~2시간) · 요약 1회 비용 1원 미만

## 11. 열린 질문

1. 시작을 **노션 DB**로 할지, 처음부터 **웹앱**으로 갈지?
2. 주 용도가 "수업자료 제작"인지 "수업 평가·연구"인지 — 무게중심에 따라 MVP 순서가 바뀜
3. 음성 번역이 실제로 자주 필요한지(다국어 수업?)
4. 갤러리 자동 인식이 꼭 필요한지, 아니면 "공유 한 번"으로 충분한지
